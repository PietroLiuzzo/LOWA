## Linked Open Data for Written Artefacts</span>

**<span lang="EN">Hamburg 26/5–28/5 2021 (Zoom)</span>**

**<span lang="EN"> </span>**

<span lang="EN">The training will introduce participants to basic Linked
Open Data technologies and show techniques to produce, store, visualize,
query, reuse, and share data as Linked Open RDF Data.  </span>

<span lang="EN">Based mainly on the experience of the Beta Maṣāḥǝft
project the training will use examples from epigraphy, codicology, and
papyrology and will welcome further diverse datasets to be
connected.</span>

##### []()<span lang="EN" style="color:black">Exercise</span>

<span lang="EN">Creating an ontology is not easy. Not because of the
data or the code, but because of the ontology. Knowledge modelling is
not easy. The data models used to represent knowledge need to be easy
and are. In this exercise we will try this out.</span>

<span lang="EN"> </span>

**<span lang="EN">Instructions</span>**

<span lang="EN">Download and install Protege [<span
style="color:#1155CC">https://protege.stanford.edu/products.php\#desktop-protege</span>](https://protege.stanford.edu/products.php#desktop-protege)
or login to use []()[<span style="color:#1155CC">Web
Protégé</span>](https://webprotege.stanford.edu) online.</span>

<span lang="EN"> </span>

<span lang="EN">Step 1. You will design your ontology from scratch. We
do not want to build the one which will be used, but the one which
represents our understanding. Examples in the wild include pizzas,
movies, cars, etc. We will model “**Books**”. Feeling Lucky? Try
"Manuscripts". Get started how you want. Imagine the user scenario you
want. A library, a book shop, your own library, a reading list... Pick
the side of representation which you want and model it. Which Classes do
you need? Which Properties to connect them? Which vocabularies to point
to? </span>

<span lang="EN">Step 2. Check out which resources you already know
(FRBR(oo), Wikidata, Dbpedia, Viaf, etc.) and look for existing
ontologies. How and what to align? Perhaps it would have been better to
start from this instead?</span>

##### []()<span lang="EN" style="color:black">Exercise</span>

<span lang="EN">Questions come before Answers. It is way more important
to be able to build a query than to read the results. In SPARQL the
result’s format is defined with the query. In this exercise you will
practice with the formal language to build your queries. You will need
to look at the data, and think of the result format. </span>

<span lang="EN"> </span>

**<span lang="EN">Instructions</span>**

<span lang="EN">First try to reproduce in the[]() [<span
style="color:#1155CC">Wikidata query
service</span>](https://query.wikidata.org/) a query from the [<span
style="color:#1155CC">examples</span>](https://www.wikidata.org/wiki/Wikidata:SPARQL_query_service/queries/examples).</span>

<span lang="EN">Now produce *your own* SPARQL query to one of the
Endpoints listed in the presentation. You may stay on Wikidata, if you
wish and you have identified relevant data for you. You may simply
modify one of the examples.</span>

<span lang="EN">Share your query [<span
style="color:#1155CC">here</span>](https://docs.google.com/document/d/1Zrdtr2wWFlFRWe7UyW5vsH2KfIc3UBQoRdn8l3goauA/edit?usp=sharing).
An example (the one below) is provided.</span>

<span lang="EN">Now try to get a map of the current location of “written
artefacts” from Wikidata.</span>

<span lang="EN">You can start from this example query</span>

<span lang="EN"> </span>

<span lang="EN" style="font-size:
8.0pt;line-height:115%">\#defaultView:Map</span>

<span lang="EN" style="font-size:
8.0pt;line-height:115%">\# Map of Codices in Wikidata at their location
(Q213924)</span>

<span lang="EN" style="font-size:
8.0pt;line-height:115%">SELECT \* </span>
